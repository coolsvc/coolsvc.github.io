# <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">本书存在的原因</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">传统软件开发已死。曾经稳固的基础如今已在我们脚下动摇。在人工智能时代，每隔几个月就会出现突破，昨日的限制变成了明日的默认设置。</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">在 HeyGen，我们不会与这种不稳定性作斗争。我们顺势而为。我们整个开发理念都围绕着顺应人工智能的进步而构建，而不是寻求不再存在的稳定技术基础。</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">这本书记录了我们如何思考、构建和取胜。它献给每一位 HeyGen 团队成员——工程师、设计师、产品经理——以及所有想加入我们的人。本书讲述了我们如何在地基不断变动的情况下工作，以及如何将这种不稳定转化为竞争优势。</font>

# <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">核心理念</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">我们的核心理念</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">快速行动，力求做到最好。乘着人工智能浪潮，拥抱研究的不确定性，提前六个月下注，打造灵活的产品，使其能够随着模型的改进而自我升级，同时又不牺牲质量。</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">根本性转变：从基础到浪潮</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">在人工智能时代，我们缺乏稳定的技术基础。每隔几个月，人工智能技术就会发生巨大的变化。模型能力未知且瞬息万变。</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">我们正处于一代人难得一遇的技术窗口期。未来 12 个月，人工智能代表着我们这一代人的战争机遇。我们有机会打造下一个谷歌或 Facebook。这个机遇现在正在爆发式增长。我们应该全力以赴。这就是大家加入 HeyGen 的原因，也是我们在这里的原因。</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">关键区别：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">我们所说的“拥抱不稳定性”，指的是底层的人工智能技术基础——模型、能力、研究突破。我们绝不接受服务正常运行时间、产品质量或用户体验的不稳定。即使底层的人工智能技术基础不断变化，我们的产品也必须保持坚如磐石的可靠性。</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">这不是什么问题，而是我们的机遇。我们要顺势而为，而不是逆流而上。</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">传统时代：</font>**

- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">建立在稳定的基础上</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">优化寿命</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">提前 12-18 个月制定计划</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">抛光，然后发货</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">顺序开发</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">人工智能时代（我们的道路）：</font>**

- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">乘着科技浪潮</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">为自动改进而构建</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">切实规划 2 个月（与模型升级周期一致）</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">船舶学习</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">平行实验</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">这与敏捷开发有何不同</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">传统敏捷假设技术相对稳定，功能可预测。而我们工作的环境是，基础技术每隔几个月就会发生变化。我们的方法扩展了敏捷原则，但针对的是技术不稳定性，而非稳定的迭代周期。</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">波浪骑士的优势</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">竞争对手追逐稳固的阵地，却被下一次飞跃所蒙蔽。我们设计的目标是随着模型的进步而自动改进，并选择顺势而为，而不是与之对抗。</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">了解哪些会变化，哪些会保持不变：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">对于我们来说，重要的是了解哪些组件在短期内可能会发生变化（模型、功能）以及哪些组件不太可能发生变化（用户工作流程、核心问题），并围绕那些不会改变的事物构建我们的产品/系统，同时顺应模型改进的浪潮。</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">乘风破浪的机会示例：</font>**

- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">情境学习 vs. 微调 vs. RAG 方法</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">语音模型的后处理改进</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">多供应商系统优化</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">质量悖论</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">我们快速行动，同时力求做到最好。这听起来似乎有些矛盾，但你若能理解：快速行动使我们能够长期打造更高品质的产品，并更快地交付客户价值。竞争对手每月只推出一项功能，而我们却能进行五次实验。我们的学习速度是竞争对手的五倍。这些学习成果最终转化为卓越的产品。</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">快速行动并不意味着快速发布功能，而是意味着快速交付客户价值（并快速学习）。速度服务于我们最终的目标：成为绝对最佳。</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">尤其对于视频内容而言，质量是不容妥协的。用户喜欢产品并非因为其界面精美——他们喜欢的是能够以卓越的质量解决他们问题的产品。我们的成功标准是：任何用户在我们平台上都能获得的平均视频质量。</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">我们的节奏：</font>**

- **<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">为期两个月的路线图规划</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">——与 AI 发展周期同步。与领导层、技术主管和产品经理深入探讨产品回顾和战略（设计可选，但鼓励参与）。理想情况下，会议最好以面对面形式进行，以最大程度地提高产出并突破干扰。会议对绩效保持完全透明和诚实的态度，并根据需要调整战略。</font>
- **<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">6-12 个月的战略押注</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">——预测并为下一个重大突破做好准备</font>
- **<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">每两周一次的承诺清单</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">——产品和工程团队共同决定并确定每个团队对特定交付成果的承诺优先级</font>
- **<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">每日发布</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">——每天都会进行改进、功能或实验</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">理念：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">短周期让我们与人工智能发展的步伐保持一致。周期足够长，足以构建有意义的产品；周期足够短，足以适应技术格局的变化。</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">进行波浪乘行实验</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">重要提示：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">该框架更多地适用于现有产品和增长领域，而不太适用于可能需要更长时间探索的新功能和研究领域。</font>

- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">构建预期变化的抽象层（警告：在尚未准备好时不要过度抽象）</font>

- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">积极地对所有内容进行版本控制</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">设计系统在模型改进时也能改进</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">技术债务的时间分配：</font>**

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">我们的原则：将减少技术债务视为对未来速度的投资。我们赞赏那些能够显著提高团队生产力和系统可靠性的技术债务减少工作。技术债务工作应该与业务成果和速度改进明确挂钩。</font>

# <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">操作原则</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">1. 速度就是一切（绝不妥协）</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">新现实：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">在人工智能时代，学习最快的团队将获胜。就是这样。</font>

- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">几天内发货，而不是几周</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">如有疑问，请进行实验</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">动力比完美更重要</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">行动缓慢是唯一不可饶恕的罪过</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">实际应用：</font>**

- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">MVP 用于测试想法，而不是最终产品</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">足以验证 > 完美但迟了</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">交付不完美的产品，然后跟进：如果它不起作用，就停止使用，或者如果用户关心，就改进它，直到它达到最佳状态</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">错误比不完美的功能更糟糕（错误阻碍学习）</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">速度是一种态度</font>**

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">速度不仅仅关乎执行力，更关乎心态。我们不会说“等到周一再发布，这样就万无一失了”。这暴露了我们缺乏紧迫感，缺乏快速学习的意愿（宝贵的数据损失了 2-3 天），缺乏责任感，以及对执行力的怀疑。这种心态阻碍了我们取得成功。成功者拥有、发布、学习和适应的速度都很快。</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">行动偏好 > 追求完美</font>**

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">如果你追求正确，那你就太慢了。不要害怕犯错，要害怕学得太慢。</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">2. 拥抱科技浪潮</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">别再追求技术稳定性了。它根本不存在。AI 基础每两个月就会发生变化。设计你的产品时，要让它随着模型的变化而自动改进。构建能够预测变化的抽象层。让你的产品体验与 AI 的进步同步发展，而不是被它左右。</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">3. 不同意但承诺</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">我们正处于战争时期，而非和平时期。每个人都会贡献想法和反馈，但决策必须迅速。一旦做出，即使意见不合，我们也要全力以赴。缺乏承诺而导致的战略缺陷，比那些我们可以迅速纠正的不完美决策更糟糕。</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">4.通过创新创造用户价值</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">用户喜欢能够解决问题的产品，而不是漂亮的界面。创新与用户喜爱紧密相连。我们利用人工智能革新了人们制作视频的方式，打造了前所未有的奇妙体验。但如果不解决实际问题，创新就毫无价值。</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">入职挑战：</font>**

- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">人工智能产品的能力因用户技能而异</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">我们的责任：教授用例，而不仅仅是功能</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">成功=任何用户都可以达到的平均质量</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">衡量视频质量，而不仅仅是视频创作</font>

## <font style="color:rgb(15, 20, 25);background-color:rgba(0, 0, 0, 0);">5. 构建与购买：一个简单的规则</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">只要能提供最佳的用户体验，我们就会去做。</font>

- **<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">内部构建：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">阿凡达视频模型（没有外部供应商符合我们的质量标准）</font>
- **<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">外部提供商：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">语音（质量足够好，资源限制）</font>
- <font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">不自私，只求结果</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">两人规则：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">原则上，由一名 PM/设计人员和一名工程师（共两人）共同构建原型。我们不会为了维护每个人的感受而进行优化，以达成共识。我们的目标是加快进度，尽快在市场上验证创意，以便我们作为一个团队能够打造出更好的产品。</font>

<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">每个人都有机会为新想法构建原型。在人工智能时代，技术上可以构建的东西无穷无尽（就像我们在黑客马拉松中所做的那样，真的很棒）。关键在于拥有一个高效的团队来推动快速决策。</font>

4. **<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">生产就绪：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(255, 255, 255, 0);">从原型到生产的每个功能都必须经过精心设计</font>
